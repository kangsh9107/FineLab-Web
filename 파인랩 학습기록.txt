프로그래머 레벨체크
: 각 단계 지식 아는 부분 확실히 정리하기
인텔리제이 깃배쉬 명령어 직접쳐보면서 알아두면 좋다
: git clone (깃 레파지토리 URL) / 깃허브 프로젝트 로컬에 복사. 컨트롤+v는 붙여넣기 안되니 마우스 우클릭하고 paste
넥서스 레파지토리란?
: 메이븐 레파지토리에 없는 내용이나 프로젝트에 맞춰 커스텀한 내용을 빌드, 관리하기 쉽게 해주는 레파지토리
깃허브 origin이란?
: origin-원격 저장소 / head-현재 위치한 작업 공간 / master-기본 브랜치
깃허브 local vs remote 차이 명확하게 알기 (pull & merge 방법)
: 로컬-내PC / 원격-깃허브 서버. 같은 파일을 수정할 경우 충돌이 생기는데 이때 stash로 임시공간을 생성해서 pull하고 merge하는 방법 사용
인텔리제이 단축키 학습할 것
: shift + shift (파일검색)
: ctrl + h (프로젝트 내 문법 등 검색)

배치 (Batch)란?
: 데이터를 실시간으로 처리하지 않고, 모아뒀다가 일괄적으로 처리하는 것
jenkins란?
: CI/CD를 도와주는 도구. (배치 간략화, 자동화 테스트, 코드 표준 준수여부 검사, 빌드 파이프라인 구성 등)
imagemagick
: 이미지 변환, 수정 등을 돕는 소프트웨어. (포맷과 상관없이 좋은 화질 출력, 메모리를 적게 사용하고 CPU를 더 사용, Exif 데이터가 소실되지 않음)
CI (Continuous integration, 지속적인 통합)
: 커밋, 빌드, 테스트를 지속적으로 시행하여 공유 레파지토리에 통합
CD (Continuous delivery, 지속적인 배포)
: 개발자의 변경 사항을 레파지토리, 클라이언트의 프로덕션 환경까지 자동으로 배포
