@Autowired / @Resource (java8까지 사용가능)
: 둘 다 의존성 주입 기능.
: @Autowired 빈 객체 찾는 순서
  > 타입이 같은 빈 객체
  > 타입이 같은 빈 객체가 두 개 이상이고, @Qualifier 어노테이션이 붙어있지 않으면 이름이 같은 빈 객체
  > @Qualifier or @Primary 어노테이션이 붙어 있는 빈 객체
: @Resource 빈 객체 찾는 순서
  > name 속성 값으로 지정한 빈 객체
  > name 속성이 없으면 타입이 같은 빈 객체
  > name 속성이 없고, 타입이 같은 빈 객체가 두 개 이상이면 이름이 같은 빈 객체
  > name 속성이 없고, 타입이 같은 빈 객체가 두 개 이상이고, 이름이 같은 빈 객체가 없으면 @Qualifier or @Primary 어노테이션이 붙어 있는 빈 객체
스네이크 케이스
: 밑줄을 공백으로 대체해서 명명. ex) Find_All_Users
@JsonProperty
: 스프링에서 DTO나 Domain 객체를 이용해 JSON의 직렬화, 역직렬화를 할 때, 응답 객체에 특정 필드를 추가하고 싶지 않은 경우 사용한다.
: 보통 서버단은 카멜, 클라이언트단은 스네이크로 표현하기 때문에 데이터의 Key가 달라져서 인식을 못하는 경우가 생길 수 있다.
@Ignore
: 해당 메서드 실행 안하기.
@Scheduled(cron = "초 분 시 일 월 요일")
: 해당 시간마다 실행. ex) "0 00 02 * * *" 매일 새벽 2시 실행. 배치에 사용한다. 요일은 0~7이며 0과 7은 일요일이다.
@Valid
: DTO 또는 객체를 검증을 돕는다.
AES (Advanced Encryption Standard)
: 대칭 키 알고리즘. 대입 치환 방식 (SPN, Substitution-Permutation Network) 으로 암호화, 복호화.

formLogin()
: Spring Security가 제공하는 인증방식으로 로그인에 사용할 때 예시는 다음과 같다.
  > 클라이언트가 Server에 URL 요청 시 해당 URL이 인증이 필요한 경우 Server는 Login 페이지를 return한다.
  > 클라이언트는 username과 password를 입력하여 로그인을 요청하면 Post mapping으로 Server에 데이터를 전송한다.
  > Server는 해당 로그인 정보를 확인하고, DB에 존재하면 Session과 Token을 생성해 저장한다.
